# Turn Log to HTML Converter

Convert turn logs (MIME multipart format) generated by `TurnLogger` into a beautiful, interactive HTML report with a clean table layout.

**Script:** `agent/turn_log_to_html.py`

## Features

✨ **Table Layout** — Clean, scannable table with # | Q/T/S | TYPE | CONTENT columns  
🎨 **Brand Colors** — Uses #0053e2 (blue) and #ffc220 (spark yellow)  
🪄 **JSON Prettifier** — Toggle between compact and formatted JSON (EXEC only)  
📊 **Compact Design** — Minimal padding, maximized content space  
📍 **All Visible** — No collapsing, everything shown by default  
🔍 **Grep-Friendly** — Source logs remain plain text for debugging  

## Usage

### Basic

```bash
python agent/turn_log_to_html.py ~/atom-agentic-ai/logs/session-abc-123/
```

This generates `session-abc-123.html` in the parent directory.

### Custom Output

```bash
python agent/turn_log_to_html.py ~/atom-agentic-ai/logs/session-abc-123/ -o my_report.html
```

### Auto-Open in Browser

```bash
python agent/turn_log_to_html.py ~/atom-agentic-ai/logs/session-abc-123/ --open
```

## Turn Log Format

Turn logs are created by `agent/turn_logger.py` and follow this naming convention:

```
q{QQ}.t{TT}.s{SS}.{type}.{label}.txt
```

Where:
- `QQ` = Query number (user prompt)
- `TT` = Turn number (model request within a query)
- `SS` = Sequence number (logged item within a turn)
- `type` = `thinking` | `text` | `plan` | `exec`
- `label` = 50-char description (auto-sanitized)

Each file contains MIME multipart content:

```
Boundary: ----=_Part_abc123...
Timestamp: 2026-04-09T20:26:27.370560+00:00
Query: 1
Turn: 1
Tool: execute_command
Call-ID: toolu_vrtx_015vZN58xeLVH27y1X9RPwgv

----=_Part_abc123...
Content-Type: input

{"command": "ls -la", "working_dir": "."}

----=_Part_abc123...
Content-Type: output

Exit Code: 0
STDOUT:
total 48
drwxr-xr-x  12 user  staff   384 Apr  9 20:26 .
...

----=_Part_abc123...--
```

## HTML Output

The generated HTML report displays entries in a compact table format:

### Table Columns

| Column | Width | Alignment | Description |
|--------|-------|-----------|-------------|
| **#** | 25px | Right | Line number (minimal padding) |
| **Q/T/S** | 55px | Right (header centered) | Query/Turn/Sequence coordinates |
| **TYPE** | 60px | Left | Color-coded entry type badge |
| **CONTENT** | Remaining | Left | Full content with auto-wrap |

### Entry Type Display

**TEXT entries** — Just the badge, clean content:
```
TEXT
I'll help you find when the `phoa` value is NOT set to `subAppId`...
```

**PLAN entries** — Badge + raw content (no labels, no prettify button):
```
PLAN
{"command": "git clone git@github.com:user/repo.git", "working_dir": "."}
```

**EXEC entries** — Badge + labeled input/output sections with prettify:
```
EXEC

input 🪄
{"command": "git clone...", "working_dir": "."}

output 🪄
Exit Code: 0
STDOUT:
Cloning into 'hubble-server'...
```

### Color-Coded Type Badges

- 💭 **THINKING** (purple) — Extended reasoning
- 🤖 **TEXT** (blue) — LLM text responses
- 📋 **PLAN** (yellow) — Tool calls LLM wants to make
- ⚡ **EXEC** (green) — Tool execution results

### Design Philosophy

- **Minimal padding** — First 3 columns use `py-1` and `pr-1` only
- **Fixed widths** — Metadata columns as narrow as possible
- **Auto-wrap content** — Text wraps in CONTENT column for readability
- **All visible** — No collapsing, full content displayed immediately
- **Consistent spacing** — Both badges and content boxes use `px-2 py-0.5`

## Why HTML?

Turn logs are great for grep/debugging, but when you need to:
- Review a long conversation flow (236+ entries!)
- Share with non-technical stakeholders
- Understand tool execution sequences
- Debug multi-turn agent loops
- See everything at once without clicking

...a clean, table-based HTML view beats navigating 200+ plain text files! 🐶
